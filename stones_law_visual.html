Can 0 = 1?

Two paths to the same violation: continuation after admissibility fails

Fake Algebraic Proof
Assume: a = b
Then: a² = ab
Subtract b² from both sides:
a² - b² = ab - b²
Factor both sides:
(a - b)(a + b) = b(a - b)
✗ CANCEL (a - b) VIOLATION
Why this fails: Since a = b, then (a - b) = 0. Canceling (a - b) is division by zero. The domain guard "a ≠ b" is required for division, but we're operating in the domain where a = b.
Result: a + b = b
↓ (All steps below are invalid)
2b = b
2 = 1
The Trap: The proof continues perfectly after the violation. Each subsequent step follows logically from the previous one. The algebra is "correct" — but the domain constraint was violated, making everything after that point meaningless.
Misused Set Identity (De Morgan)
De Morgan's Law (Correct):
A - (B ∪ C) = (A - B) ∩ (A - C)
This is valid ONLY when the set A is preserved in both intersections. The guard "A" ensures we're only working with elements that were originally in A.
Rewrite using complement notation:
A - X = A ∩ XC
Apply to both sides:
A ∩ (B ∪ C)C = (A ∩ BC) ∩ (A ∩ CC)
✗ DROP GUARD A VIOLATION
A ∩ (BC ∩ CC)
Why this fails: The set A was the domain constraint. Removing it means the operation (BC ∩ CC) is now treated as a global law that applies everywhere, not just within the boundary of A.

This is equivalent to the algebraic case: we removed the guard that made the operation valid.
Treated as global identity:
(BC ∩ CC)
↓ (Domain boundary dissolved)
The Trap: Unlike the algebra example, this violation is harder to see because set notation is less familiar. But it's the exact same pattern:
  • Start with a valid operation under constraints
  • Drop the constraint that made it valid
  • Continue as if nothing changed
  • Arrive at contradiction
⟷ Continuation after admissibility fails ⟷
Both proofs share the same structural failure
✓ CORRECT
(a - b) cancellation
Domain: a ≠ b
Operation: Division by (a - b)
Valid: Yes, (a - b) ≠ 0
✗ VIOLATION
(a - b) cancellation
Domain: a = b
Operation: Division by (a - b) = 0
Valid: NO — divided by zero
✓ CORRECT
A ∩ (BC ∩ CC)
Domain: A present
Operation: Intersection within A
Valid: Yes, scoped to A
✗ VIOLATION
(BC ∩ CC)
Domain: A removed
Operation: Global law claimed
Valid: NO — guard removed
Admissibility Rule (Stone's Law)
An operation is only valid within its domain.
If the condition is violated, the derivation must stop.

The pattern is universal: Remove the constraint that makes an operation valid, continue as if nothing changed, and the system becomes incoherent — even though each individual step appears correct. This is why admissibility gates must be enforced before execution, not after. Once the boundary is crossed, no amount of coherent reasoning can recover validity.

0 ≠ 1 is not a moral principle. It's a type constraint.
